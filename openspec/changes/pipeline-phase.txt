Phase 1（基础设施）─────────────────────────
    1. pipeline-data-model
       · Template/Run/Stage/Event/Gate 的完整类型
       · 持久化文件布局（runs/<id>/stages/<id>/events.jsonl）
       · templateSnapshot 机制
       · 5 级 severity + gate 阈值

    2. pipeline-orchestrator-architecture
       · Fyllo Orchestrator agent 架构（双层 agent）
       · Tool 集合定义（spawn_stage_subagent / evaluate_gate / ...）
       · 模型策略（轻量 orchestrator + 模板隐含强模型）
       · 状态机（Run/Stage 状态转移）
       · 退出拦截

  Phase 2（UI 落地）─────────────────────────
    3. pipeline-run-detail-view
       · 横向 Stage 流 + 自动定位活动 stage
       · 5 种 stage 详情面板（Discuss/Code/Test/Review/Deploy）
       · Summary / Agent Log 双视图
       · Orchestrator 折叠抽屉

    4. pipeline-template-editor
       · 表单编辑器 + YAML 只读预览
       · 上下箭头排序
       · Stage 配置（包含 review severity 阈值）
       · 内置模板复制机制

    5. pipeline-new-run-flow
       · 创建 Run 对话框
       · 输入源选择（manual / 已 connected integration）
       · 模板选择 + 默认模板

  Phase 3（人工介入闭环）─────────────────────
    6. pipeline-human-intervention
       · Gate 审批 UI（manual approval）
       · Fyllo clarification 响应 UI
       · Sidebar 状态指示器（4 种视觉状态）
